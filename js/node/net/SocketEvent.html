Enumeration of events for Socket objects.

Fields

inline read only Close:SocketEvent<Bool ‑> Void> = cast "close"

Emitted once the socket is fully closed. The argument had_error is a boolean which says if the socket was closed due to a transmission error.

Listener arguments:

had_error - true if the socket had a transmission error

inline read only Connect:SocketEvent<Void ‑> Void> = cast "connect"

Emitted when a socket connection is successfully established. See Socket.connect.

inline read only Data:SocketEvent<EitherType<Buffer, String> ‑> Void> = cast "data"

Emitted when data is received. The argument data will be a Buffer or String. Encoding of data is set by Socket.setEncoding.

Note that the data will be lost if there is no listener when a Socket emits a 'data' event.

inline read only Drain:SocketEvent<Void ‑> Void> = cast "drain"

Emitted when the write buffer becomes empty. Can be used to throttle uploads. See also: the return values of Socket.write

inline read only End:SocketEvent<Void ‑> Void> = cast "end"

Emitted when the other end of the socket sends a FIN packet.

By default (allowHalfOpen == false) the socket will destroy its file descriptor once it has written out its pending write queue. However, by setting allowHalfOpen == true the socket will not automatically end its side allowing the user to write arbitrary amounts of data, with the caveat that the user is required to end their side now.

inline read only Error:SocketEvent<Error ‑> Void> = cast "error"

Emitted when an error occurs. The 'close' event will be called directly following this event.

inline read only Lookup:SocketEvent<Null<Error> ‑> String ‑> DnsAddressFamily ‑> Void> = cast "lookup"

Emitted after resolving the hostname but before connecting. Not applicable to UNIX sockets.

inline read only Timeout:SocketEvent<Void ‑> Void> = cast "timeout"

Emitted if the socket times out from inactivity. This is only to notify that the socket has been idle The user must manually close the connection. See also: Socket.setTimeout
